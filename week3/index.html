Objects

  • Objects are like a dictionary and can hold anything, primitive data, functions, other objects (such as arrays)
  • created by wrapping the data in {}, each property can have a key-value pair, and are separated by commas
  • Objects can be defined literal (i.e. const spiderman = {};) 0r with constructor function (i.e. const spiderman = new Object();
  • Access property values using dot notation: superman.propertyName
  • OR square brackets: superman['propertyName'] (this is the only way to access values not named using proper naming rules, like ones with a space in them)
    You can also concatenate the property names within the brackets if needed
  • Symbols can be used as a property key, which reduces the chance of property redundancy
