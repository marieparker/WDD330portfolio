Objects

  • Objects are like a dictionary and can hold anything, primitive data, functions, other objects (such as arrays, functions, JSON, basically anything contained within {})
  • created by wrapping the data in {}, each property can have a key-value pair, and are separated by commas
  • Objects can be defined literal (i.e. const spiderman = {};) 0r with constructor function (i.e. const spiderman = new Object();
  • Access property values using dot notation: superman.propertyName
  • OR square brackets: superman['propertyName'] (this is the only way to access values not named using proper naming rules, like ones with a space in them)
    You can also concatenate the property names within the brackets if needed
  • Symbols can be used as a property key, which reduces the chance of property redundancy
  • Calling Methods (functions)
    • example: superman.functionName()
    • OR example: superman['functionName']()
    • DON'T FORGET ()
  • Checking for Properties/Methods
    • in operator
      example: 'keyName' in superman;
    • Using Logic
      example: superman.keyName !== undefined (if it returns true then the key exists, false it doesn't)
    • hasOwnProperty: this is useful to find out if an object has it's own property or if it is inherited from another object
      example: superman.hasOwnProperty('keyName');
  • to iterate over all keys or values or entries:
    • Object.keys()
    • Object.values()
    • Object.entries()
  • Adding/Changing Properties: dot method
    example: superman.city = 'Metropolis';
  • Deleting Properties:
    example: delete superman.keyName;
  • Objects can contain other objects (see main.js for the main page of this website for an example)
  • Objects as parameters
    function greet({greeting,name,age}) {
    return `${greeting}! My name is ${name} and I am ${age} years old.`;
    }

    greet ({ greeting: `What's up, dude?`, age: 10, name: `Bart});
    (order doesn't matter when using an object to pass parameters because it is based on the key name)
  • You can also set default parameters using this:
    function greet({greeting='Hello',name,age=18}) {
    return `${greeting}! My name is ${name} and I am ${age} years old.`;
    }
    (These defaults are overridden if another value is passed through the parameters when the function is called)
  • this refers to an object when a method stored inside that object is referring to the object
    example:
    const dice = {
    sides: 6,
    roll() {
    return Math.floor(this.sides * Math.random()) + 1;
    }
    }
    (random() and floor() are Math methods to return a number bewtwwen 1 and the number of sides)
  • Object Literal Pattern is a useful way to store related data or functions to reduce the cance of variables being named the same things and accidentally changed. It creates a nameSpace (another way to define it)
    example: a function woudl no longer just be named square(), it would be called myObjectName.square()
  • JSON
    • Property names must be doubel quoted
    • Values can include: double-quoted strings, numbers, true, false, null, arrays, and objects
    • NO functions allowed
    • Methods:
      • .parse() turns JSON format into JS object
      • .stringify() turns a JS object in JSON (this will just ignore any function values)
      • Useful when using Ajax requests, or using localStorage
  • MATH object: a built-in object representing certain math constants (like PI, LN, E, SQRT2, etc...), and methods. These properties CANNOT be changed (immutable)
      Math Methods:
    • Math.abs-absolute value
    • Math.ceil-rounds up to next integer
    • Math.floor-rounds down to next integer
    • Math.round - rounds to nearest integer
    • Math.exp-exponent
    • Math.pow-to the pow
    • for a complete list see here
  • Math.random() generates a random number between 0 and 1. To get a higher number, simplay multiply by that number
    example: 6 * Math.random();
    If you want a random integer use Math.floor()
    example: Math.floor(6 * Math.random());
  • DATE object: creates a new object each time because the date, or time, or some property of the object will be different each time it is created
    but you can also create an object of a specific date by passing parameters to the constructor
    • example: const today = new Date();
    • .toString(): converts the whole object into a string (this methid can be used on ANY object, not just Date)
    • still starts counting at 0, so January is '0' month
    • The Epoch is January 1, 1970: a timestamp is measured by how many milliseconds since the epoch
    • Getter Methods
      • .getDay(), or getUTCDay(): finds the day of the week: 0 for Sunday, 1 for Monday, etc...
      • getDate(), or getUTCDate(): finds the day of the month
      • .getMonth(), or getUTCMonth(): finds the month, 0 for January, 1 for February, etc...
      • .getFullYear(), or getUTCFullYear(): finds the year
      • .getHours(), .getUTCHours(), .getMinutes(), etc...
      • .getTime(): returns the number of milliseconds since the Epoch
        This if usefull for going by certain increments of time (i.e. 1000 * 60 * 60 * 24 represents one day)
    • Setter Methods
      • Change the values held in the Date object
      • .setDate(), .setMonth(), etc...
  • RegExp Objects
    • example: const pattern = /[a-zA-Z]+ing$/;
    • OR const pattern = new RegExp('[a-zA-Z]+ing'); (this constructor method makes it so you can creates expressions using strings(like when using user input), and you can also use a variable
    • Methods
      • .test('testString'): returns true or false, used to see if this expression 'testString' matches the regular expression pattern
      • .exec(): similar to test, but returns an array with the first match found
      • Character Groups are placed inside brackets [], this is where the dashes used in ther previous examples come in
    • Properties
      • example: pattern = /java/i;
      • global (g) returns ALL matches
      • ignoreCase (i) makes the pattern case-insensitive
      • multiline (m) searches all lines
    • Other things
      • Special Characters
      • Modifiers can be placed after a token to deal with multiple occurences
        example: * matches one or more occurences of that token
        $ marks the position immediately after the last character in the string(
